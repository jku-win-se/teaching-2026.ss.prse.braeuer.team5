# Integration Readiness

## Goal

Prepare work for review and merge without losing context.

## Before PR or Merge

- ensure the plan artifact reflects the final or current state
- summarize what changed
- record remaining risks
- identify follow-up work if the issue is not fully complete
- make sure commit messages follow repository convention

## Review Support

A reviewer should be able to understand:

- what issue this work addresses
- why this approach was chosen
- what was verified
- what is still open
