# Units of Work

## Goal

Break the issue into small, reviewable steps.

## Rules

- prefer steps that can be inspected independently
- avoid mixing architecture, implementation, and cleanup in one jump
- keep the next step concrete
- if the issue is large, define an ordered sequence

## Good Unit Examples

- create the directory structure for the workflow
- add one shared workflow file
- add one personal plan file
- update one template to match the workflow

## Avoid

- "implement the whole feature"
- "set up everything at once" without checkpoints
