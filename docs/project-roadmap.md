# Projekt-Roadmap

## Ziel

Diese Datei enthaelt nur den groben Projektfahrplan. Die eigentliche Arbeitsplanung liegt in GitHub-Issues und Milestones.

## Termine

- Prototype: 20.03.2026
- Release 1: 14.04.2026
- Release 2: 05.05.2026
- Release 3: 26.05.2026
- Abschlusspraesentation: 19.06.2026
- Projektabgabe: 26.06.2026

## Planungsregel

- Dauerhafte Projektdokumente bleiben im Repository.
- Konkrete Arbeitspakete werden als GitHub-Issues gepflegt.
- Die Releases werden ueber GitHub-Milestones geplant.

## Umsetzung pro Release

- Die konkrete Arbeit fuer jedes Release wird in GitHub-Issues geplant.
- Die Teammitglieder erstellen und pflegen diese Issues selbst.
- Das Repository enthaelt nur den groben Fahrplan und keine statische Aufgabenliste.
