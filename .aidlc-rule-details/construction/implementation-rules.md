# Implementation Rules

## Goal

Implement only the approved next unit of work.

## Rules

- do not skip directly from analysis to broad implementation
- change only the files needed for the current step
- keep naming aligned with repo conventions
- preserve existing team documentation unless the issue requires updates
- update the plan artifact after meaningful progress

## For This Repository

- React, TypeScript, and Vite files should stay consistent with current project style
- Supabase-related behavior must not be fabricated
- placeholder code should only be replaced when the issue actually covers that scope
