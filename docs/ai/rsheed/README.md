# Rsheed

Persoenlicher Arbeitsbereich fuer optionale KI-Artefakte.

Empfohlenes Muster pro Issue:

- `issue-<nummer>-seed.md` fuer Ziel, offene Fragen, Fortschritt und naechste Schritte
- weitere Dateien nur bei Bedarf, z. B. fuer einen groesseren Plan oder eine Architekturentscheidung

Beispiel:

- `issue-60-seed.md`
- `issue-30-seed.md`

Zusaetzliche persoenliche Arbeitsregel:

- Bevor die KI etwas direkt in Supabase aendert, muss sie vorher genau erklaeren:
  - was konkret geaendert werden soll
  - warum die Aenderung noetig ist
  - was die fachliche und technische Auswirkung fuer den Nutzer ist
- Erst nach dieser expliziten Erklaerung und Zustimmung darf eine Supabase-Aenderung ausgefuehrt werden.
