# Requirements Analysis

## Goal

Turn the GitHub Issue into a clear, bounded engineering task.

## Steps

1. Read the issue carefully
2. Identify explicit goals
3. Separate scope from non-goals
4. Extract or refine acceptance criteria
5. Note missing context
6. Identify risk areas

## Repository Questions

- Which screens, components, routes, or config files are affected?
- Does the issue imply a data model change?
- Does the issue depend on auth or roles?
- Are there existing placeholders that should be replaced instead of adding duplicates?

## Output

Update the plan artifact with:

- goal
- scope
- acceptance criteria
- open questions
- assumptions
