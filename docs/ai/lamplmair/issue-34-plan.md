## Referenz
FR-10 Regel-Engine

## User Story
Als Benutzer moechte ich Regeln definieren, damit mein System automatisch auf Ereignisse reagiert.

## Background
Regeln erweitern die reine Zeitsteuerung um Bedingungen und Aktionen. Sie sind ein zentraler Produktbaustein.

## Akzeptanzkriterien
- Definition von Ausloeser und Aktion ist moeglich.
- Regeln werden automatisch ausgefuehrt.
- Mehrere Regeln sind moeglich.

## Implementation Hints
- Regelmodell klar in Trigger und Aktion trennen.
- Fehler bei der Ausfuehrung im UI sichtbar machen.
